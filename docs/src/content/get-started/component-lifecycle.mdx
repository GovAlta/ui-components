<goa-text size="heading-xl" mt="none" mb="m">Component lifecycle</goa-text>
<goa-text size="body-l" mt="none" mb="2xl">The component lifecycle defines the stages each Design System component moves through, from Experimental to Production to Legacy.</goa-text>

<p>
  To request a new component or propose an enhancement, follow the
  <a href={withBase(`/get-started/contribute`)}>design contribution process</a>.
</p>
<goa-text as="h2" id="experimental" size="heading-m">Experimental</goa-text>
<p>
  Experimental components are new or significantly updated. They are available for testing and feedback
  before public release.
</p>

<goa-text size="heading-xs">Characteristics</goa-text>
<ul>
  <li>Feedback is gathered from selected teams</li>
  <li>May be unstable or include breaking changes</li>
  <li>Limited documentation</li>
</ul>
<goa-text as="h2" id="production" size="heading-m">Production</goa-text>
<p>
  Production components are stable and ready for broader adoption.
</p>

<goa-text size="heading-xs">Characteristics</goa-text>
<ul>
  <li>Fully documented, including design guidelines, properties, and examples</li>
  <li>Meet accessibility and design system standards</li>
  <li>Optimized for mobile use</li>
  <li>Actively maintained with updates and bug fixes</li>
  <li>Publicly available documentation</li>
</ul>
<goa-text as="h2" id="legacy" size="heading-m">Legacy</goa-text>
<p>
  Legacy components are older components maintained primarily to support older implementations. If
  possible, transition to production is recommended.
</p>

<goa-text size="heading-xs">Characteristics</goa-text>
<ul>
  <li>No active maintenance or updates</li>
  <li>Critical issues may remain unresolved</li>
  <li>Recommended alternative clearly identified</li>
  <li>Marked as "Legacy" in official documentation</li>
</ul>
