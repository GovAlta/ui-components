Bug #3643: Popover overflows right edge when trigger is near viewport boundary View on GitHub The Popover rewrite switched to CSS Anchor Positioning, which always left-aligns the popover to the trigger via inset-inline-start: anchor(left). When the trigger is near the right edge of the viewport, the popover content extends off-screen. The fix right-aligns the popover when the trigger is in the right half of the viewport so it stays within the viewport.
