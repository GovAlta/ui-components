Bug #4009: Checkbox and radio item labels have inconsistent weight and color The checkbox label and the radio item label rendered differently even though they are the same kind of form control label. The checkbox had the right color but regular weight, and the radio had the right medium weight but no color, so it inherited the near black page default. After the fix both render in medium weight and resolve their color through component tokens that point at the shared secondary input text color. Test Cases Test 1: Default size, side by side The checkbox and radio item labels should be identical in weight (medium) and color (secondary input text color). Test 2: Compact size The compact checkbox label keeps its smaller font size but stays medium weight. Test 3: Disabled state Disabled labels use their own disabled color token and are unaffected by this change.
